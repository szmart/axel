The "rich content" plugin

Semantic example

  • noedit (true|false) : set it to true to make the field read only (constant).
  • class (name) : set a class name on the handle.
  • lang (fragments|html|semantic) : set the input/output format (default is fragments).
  • multilines (normal|none) : allows for line breaks in the editable content.

The `semantic' configuration accepts an input composed of a list of custom, user-chosen tags, supposedly representing some categories of meaning. The semantic tags can match CSS classes with the same names, and will be displayed in accordance with the definition of the classes. Non meaningful nodes can be created with the Text, and Link tags. The latter should assume the same structure as the links of the `fragments' configuration, with a `LinkRef' subnode containing the target, and a subnode containing some semantic tag for the texte (or the non-meaningful Text tag).
